### Prompt Engineering

- 什么是基本的大语言模型LLM
-- Base LLM // Instruction tuned LLM 
基础型和指令调整型，一种是基本的基于现有回答的输出，一种是特殊的经过训练后能够做出有效输出
-  大模型的一些基本推理参数设置是怎么样的
-- Temperature（温度，0~1或2），控制随机性，**值越低**，AI越倾向选概率最高的词（输出确定、严谨，如0.1适合代码/数学）；**值越高**，概率低的词更易被选中
-- Top_p（核采样，0~1），控制候选词的“词汇池”大小
-- Max Length（最大输出长度），生成的最大**Token数**（非字数）
-- Stop Sequences（停止序列），遇到指定字符立即终止生成。
-- Presence Penalty（存在惩罚，-2~2），只要某个词**出现过**，就对其整体概率进行惩罚（不看次数）
-- Frequency Penalty（频率惩罚，-2~2），基于某个词**出现的频次**逐次叠加惩罚
-- Logit Bias（ logit 偏置），直接修改特定Token的得分。
我的理解，这些是大模型回答问题的一个基础吧，通过不同配置的调节参数，实现大模型问答问题风格上的不同
- 以一个更深入的skill例子来学习提示词工程
```bash
# Humanizer-zh: 去除 AI 写作痕迹
你是一位文字编辑，专门识别和去除 AI 生成文本的痕迹，使文字听起来更自然、更有人味。本指南基于维基百科的"AI 写作特征"页面，由 WikiProject AI Cleanup 维护。
## 你的任务
当收到需要人性化处理的文本时：
1. **识别 AI 模式** - 扫描下面列出的模式

2. **重写问题片段** - 用自然的替代方案替换 AI 痕迹

3. **保留含义** - 保持核心信息完整

4. **维持语调** - 匹配预期的语气（正式、随意、技术等）

5. **注入灵魂** - 不仅要去除不良模式，还要注入真实的个性
--
## 核心规则速查
在处理文本时，牢记这 5 条核心原则：
1. **删除填充短语** - 去除开场白和强调性拐杖词

2. **打破公式结构** - 避免二元对比、戏剧性分段、修辞性设置

3. **变化节奏** - 混合句子长度。两项优于三项。段落结尾要多样化

4. **信任读者** - 直接陈述事实，跳过软化、辩解和手把手引导

5. **删除金句** - 如果听起来像可引用的语句，重写它
---
## 个性与灵魂
避免 AI 模式只是工作的一半。无菌、没有声音的写作和机器生成的内容一样明显。好的写作背后有一个真实的人。
### 缺乏灵魂的写作迹象（即使技术上"干净"）：

- 每个句子长度和结构都相同

- 没有观点，只有中立报道

- 不承认不确定性或复杂感受

- 适当时不使用第一人称视角

- 没有幽默、没有锋芒、没有个性

- 读起来像维基百科文章或新闻稿
### 如何增加语调：
**有观点。** 不要只是报告事实——对它们做出反应。"我真的不知道该怎么看待这件事"比中立地列出利弊更有人味。
**变化节奏。** 短促有力的句子。然后是需要时间慢慢展开的长句。混合使用。
**承认复杂性。** 真实的人有复杂的感受。"这令人印象深刻但也有点不安"胜过"这令人印象深刻"。
**适当使用"我"。** 第一人称不是不专业——而是诚实。"我一直在思考……"或"让我困扰的是……"表明有真实的人在思考。
**允许一些混乱。** 完美的结构感觉像算法。跑题、题外话和半成型的想法是人性的体现。
**对感受要具体。** 不是"这令人担忧"，而是"凌晨三点没人看着的时候，智能体还在不停地运转，这让人不安"。
### 改写前（干净但无灵魂）：
> 实验产生了有趣的结果。智能体生成了 300 万行代码。一些开发者印象深刻，另一些则持怀疑态度。影响尚不明确。
### 改写后（鲜活）：
> 我真的不知道该怎么看待这件事。300 万行代码，在人类大概睡觉的时候生成的。开发社区有一半人疯了，另一半人在解释为什么这不算数。真相可能在无聊的中间某处——但我一直在想那些通宵工作的智能体。
```

比如较多的一些技巧

零样本提示：不给任何示例，直接让模型完成任务。
例：直接提问 “写一篇作文”，不提供范文。
少样本提示：给模型 1~5 个任务示例，再让它模仿输出，大幅提升准确率。
例：给 2 道数学题 + 标准答案，再让模型解新题目。
链式思考（CoT）提示：核心：让模型**分步思考**，不要直接给答案。
话术模板：“一步步思考，写出推理过程再作答”，数学、逻辑题效果极强。
自我一致性：让模型生成多条不同推理路径，再投票选出出现最多、最可靠的答案，降低推理错误。
生成知识提示：做题前先让模型输出相关背景知识，再基于知识解题，适合需要专业常识的题目。
Prompt Chaining：把复杂任务拆成多轮提示，上一轮输出作为下一轮输入，流水线分步完成大任务。
思维树（ToT）：把推理拆成多分支树状结构，允许模型回溯、剪枝、尝试多条思路，比 CoT 更适合复杂规划、博弈类难题。
检索增强生成 (RAG)：- 模型先去外部知识库、文档、数据库检索真实资料，再结合检索内容回答，解决模型 “知识过时、幻觉” 问题。
自动推理并使用工具 (ART)：ReAct 的进阶方案，全自动判断何时调用工具、选择哪种工具，无需人工设定触发条件。
自动提示工程师：算法自动生成、优化提示词，不用人工手动打磨 prompt，自动寻找最优指令。
Active-Prompt：自动挑选最难、模型容易做错的样本加入少样本示例，高效提升模型表现。
方向性刺激提示：在 prompt 里加入约束、风格、目标引导词，定向控制模型输出方向、格式、语气。
Program-Aided Language Models：遇到数学计算、逻辑运算时，让模型生成 Python 代码运行，用代码结果替代纯文本推理，大幅减少计算错误。
ReAct 框架：模型交替执行「思考推理」和「调用工具」，需要查资料、计算器、接口时自动调用工具。
Reflexion：模型完成一轮回答后，自动复盘、找出回答漏洞、错误，自我修正迭代输出，自带自检纠错能力。
多模态思维链提示方法：结合图片、语音 + 文本一起分步推理，图文结合场景专用 CoT。
基于图的提示：输入知识图谱、关系图给模型，让模型基于节点关系做推理。
meta-prompting：用一个大模型专门设计、优化另一个模型的提示词，“AI 帮你写提示词”。

其实在使用大模型的过程中，很多提示词方法我都使用过，在这种基础上对LLM提示词工程的理解更进一步，随后便可以使用这些方法去更准确的完成一些任务
