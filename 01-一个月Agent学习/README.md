## 🎯 一个月后的目标

| 维度 | 目标 |
|------|------|
| ✅ **概念** | 能用自己的话讲清 Agent 的核心循环、主流范式、关键瓶颈 |
| ✅ **动手** | 能基于 OpenAI API 从零手写一个带 ReAct 循环 + 工具调用的 Agent |
| ✅ **框架** | 会用 LangGraph / AutoGen 搭建简单多 Agent 应用 |
| ✅ **项目** | 完成一个完整的 Agent 项目（如智能助手 / 代码分析工具） |

## ⚙️ Week 2：核心 — 手写一个 Agent

**目标**：亲手写出 ReAct 循环，真正理解 Agent 的内核。

> 🐍**进阶预警**：这周开始涉及实际代码，需要理解函数定义、JSON 处理、HTTP 请求、异常处理。如果卡住很正常，每个知识点我都会后续补充。

### 📖 阅读安排

| 天 | Hello-Agents | ADP |
|---|-------------|-----|
| Day 1-2 | **Ch4 智能体经典范式构建** ⭐<br>• ReAct 模式详解<br>• Plan-and-Solve<br>• Reflection | **Ch4 反思**<br>• 自我反思模式<br>• 反思 vs 普通反馈 |
| Day 3-4 | 继续 Ch4（已有代码，跟着敲）<br>• 配 API Key<br>• 跑通 ReAct 示例 | **Ch5 工具使用** ⭐<br>• Function Calling 原理<br>• 工具定义的最佳实践 |
| Day 5-6 | **Ch7 构建你的 Agent 框架** ⭐<br>• 从零搭建框架骨架 | **Ch6 规划**<br>• 任务分解<br>• 动态规划 vs 静态规划 |
| Day 7 | 复盘 + 写笔记 | 回顾 + 整理代码 |

### 🐍 Python 需要关注的点

| 知识点 | 出现在哪 | 难度 |
|--------|---------|------|
| JSON Schema 定义 | Ch4 ReAct | 🐍**进阶** — 理解工具描述的格式 |
| 解析 LLM 返回的 tool_calls | Ch4 | 🐍**进阶** — 需要处理嵌套 JSON |
| 异常捕获与重试 | Ch7 框架 | 🐍**进阶** — try/except + 循环控制 |
| 子进程执行代码 | Ch4 execute_python | 🐍**进阶** — subprocess 模块 |
| 对话历史管理 | Ch7 | 🐍**基础** — 列表追加即可 |

### 📝 需要写的笔记（4-5 篇）

1. `ReAct循环.md` — 手画循环图，标注每个步骤
2. `Function Calling机制.md` — 工具定义、解析、执行流程
3. `从零搭建框架笔记.md` — 记录你的框架结构
4. `Week2复盘.md` — 遇到的坑 + 心得

### ✅ 检验标准

- [ ] 能在本地跑通一个带 ReAct 循环的 Agent（哪怕是调用 API 算个数学题）
- [ ] 理解 Prompt 就是 Agent 的"意识"
- [ ] 理解上下文窗口是第一瓶颈
- [ ] 能说清 Function Calling 的数据流：Schema → LLM → tool_calls → 执行 → 结果返回

## 💡 重要提醒

### 遇到卡住怎么办

```
卡住 → 把问题记在笔记的「❓ 疑问」里 → 继续往下走
     → 第二遍回来看，很多问题自然就通了
     → 还是不懂？标记好，我后续点评时会重点补充
```

### 每天建议时间分配

```
40% 阅读教材
30% 动手（敲代码/画图/做笔记）
20% 写笔记
10% 回顾
```

### 不要做的事

- ❌ 不要试图读完每一章的全部细节 — 抓住核心概念和动手代码
- ❌ 不要在 Week1 就纠结 MCP 或 RL — 那是 Week3-4 的事
- ❌ 不要怕代码报错 — 报错是最好的学习机会
- ❌ 不要为了赶进度跳过动手环节 — 只看不写等于没学
