# 🐍 01 — 函数定义与类型标注

## 代码示例

```python
def get_weather(city: str) -> str:
    """
    通过调用 wttr.in API 查询真实的天气信息。
    """
    url = f"https://wttr.in/{city}?format=j1"
```

## 逐行拆解

| 部分 | 含义 |
|------|------|
| `def get_weather` | 定义一个名为 `get_weather` 的函数 |
| `city: str` | 参数 `city` 的类型是 **字符串**（类型标注，仅提示用，不影响运行） |
| `-> str` | 返回值类型也是 **字符串** |
| `""" ... """` | **文档字符串**（docstring），说明函数的作用 |
| `f"..."` | **f-string**，可以在字符串里嵌入 `{变量}` |

## 类型标注速查

```python
def hello(name: str, age: int, scores: list[float]) -> dict[str, int]:
    """参数和返回值都标清楚"""
    return {"name": name, "count": len(scores)}
```

| 写法 | 含义 |
|------|------|
| `name: str` | 参数是字符串 |
| `age: int` | 参数是整数 |
| `scores: list[float]` | 参数是浮点数列表 |
| `-> dict[str, int]` | 返回值是字典，键为字符串，值为整数 |
| `-> None` | 没有返回值 |
| `-> str \| None` | 可能返回字符串，也可能返回 None |

## 要点

> - 类型标注**不会报错**，即使你传了 `int` 但标注了 `str`，Python 也不会阻止你
> - 但它让代码**自文档化**，别人（和未来的你）一眼就知道这个函数要什么参数
> - f-string 是最推荐的字符串格式化方式，比 `"xx %s" % var` 和 `"{}".format(var)` 都更清晰
