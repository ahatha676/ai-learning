# 🐍 04 — 环境变量与配置管理

## 为什么需要环境变量？

API Key（如 OpenAI Key、Tavily Key）是**敏感信息**，不应该写死在代码里。如果代码上传到 GitHub，Key 就泄露了。

## 两种方式对比

### ❌ 方式一：直接写在代码里（不安全）

```python
API_KEY = "sk-xxx..."  # 如果上传 GitHub 就泄露了
```

### ✅ 方式二：从环境变量读取

```python
import os

api_key = os.environ.get("TAVILY_API_KEY")
if not api_key:
    return "错误:未配置TAVILY_API_KEY环境变量。"
```

## 配置 vs 环境变量

本项目使用 `config.py` 集中管理配置：

```python
# config.py — 所有配置集中在这里
API_KEY = "YOUR_API_KEY"    # 改这里就行
BASE_URL = "YOUR_BASE_URL"
MODEL_ID = "YOUR_MODEL_ID"

# 第三方服务的 Key 通过环境变量设置
os.environ["TAVILY_API_KEY"] = "YOUR_TAVILY_API_KEY"
```

其他文件引用配置时只需要 `from config import API_KEY`。

## os.environ 用法

| 写法 | 含义 |
|------|------|
| `os.environ.get("KEY")` | 获取环境变量，不存在返回 `None` |
| `os.environ.get("KEY", "默认值")` | 不存在时返回默认值 |
| `os.environ["KEY"]` | 直接取值，不存在会**报错** |
| `os.environ["KEY"] = "值"` | **设置**环境变量（只对当前进程生效） |

## 进阶：.env 文件

当配置项越来越多时，可以用 `python-dotenv` 库把 Key 放在 `.env` 文件中（不提交到 GitHub）：

```bash
# .env 文件
TAVILY_API_KEY=tvly-xxx...
OPENAI_API_KEY=sk-xxx...
```

```python
# 在 config.py 中加载
from dotenv import load_dotenv
load_dotenv()  # 自动读取 .env 文件
```

## 要点

> - 永远不要把 API Key 直接写在代码里
> - 用 `config.py` 集中管理所有配置，改一处就行
> - `os.environ.get("KEY")` 比 `os.environ["KEY"]` 更安全（取不到时返回 None 而不是崩溃）
