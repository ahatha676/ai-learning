# 🐍 07 — 类与面向对象基础

## 为什么需要类？

当代码需要**封装数据和操作**时，类比函数更好用。比如 LLM 客户端需要同时保存 `model`、`api_key`、`base_url` 三个配置，每次调用时都要用它们。

用函数需要每次都传参数：

```python
# 函数方式：每次都要传三个参数
def generate(model, api_key, base_url, prompt):
    ...

generate("gpt-4", "sk-xxx", "https://xxx", prompt1)
generate("gpt-4", "sk-xxx", "https://xxx", prompt2)
```

用类可以把配置**存起来**，每次调用只需传变化的部分：

```python
# 类方式：配置只传一次
client = OpenAICompatibleClient(model="gpt-4", api_key="sk-xxx", base_url="https://xxx")
client.generate(prompt1)
client.generate(prompt2)
```

## 本项目的类

```python
class OpenAICompatibleClient:
    """
    一个用于调用任何兼容OpenAI接口的LLM服务的客户端。
    """
    def __init__(self, model: str, api_key: str, base_url: str):
        """构造函数：初始化时把配置保存到实例"""
        self.model = model
        self.client = OpenAI(api_key=api_key, base_url=base_url)

    def generate(self, prompt: str, system_prompt: str) -> str:
        """普通方法：每次调用 LLM"""
        messages = [
            {'role': 'system', 'content': system_prompt},
            {'role': 'user', 'content': prompt}
        ]
        response = self.client.chat.completions.create(
            model=self.model,
            messages=messages,
            stream=False
        )
        return response.choices[0].message.content
```

## 类的核心概念

### `__init__` — 构造函数

在创建实例时自动调用，用于初始化数据。

```python
# __init__ 自动运行，把参数保存到 self 上
client = OpenAICompatibleClient(model="gpt-4", api_key="sk-xxx", base_url="...")
# 等价于：
# client.model = "gpt-4"
# client.client = OpenAI(api_key="sk-xxx", base_url="...")
```

### `self` — 实例本身

类内部所有方法的第一参数都是 `self`，它指向**当前实例**。

```python
def generate(self, prompt, system_prompt):
    # self.model 就是上面 __init__ 里保存的 model
    # self.client 就是上面 __init__ 里保存的 client
    ...
```

### 使用方式

```python
# 1. 创建实例（调用 __init__）
llm = OpenAICompatibleClient(
    model="gpt-4",
    api_key="sk-xxx",
    base_url="https://api.openai.com"
)

# 2. 调用方法（self 自动传入，不需要写）
llm.generate("你好")  # self=llm, prompt="你好"
```

## 类 vs 函数：什么时候用哪个？

| 场景 | 推荐方式 | 原因 |
|------|---------|------|
| 一个函数做一件事 | 函数 | 简单、直观 |
| 需要记住状态/配置 | 类 | 配置只初始化一次 |
| 需要多个相关操作共享数据 | 类 | 数据在 `self` 上共享 |
| 一次性的计算 | 函数 | 用类反而啰嗦 |

## 要点

> - `__init__` 不是必须的，但如果类需要初始化数据，它是标准写法
> - 类里方法的第一个参数必须是 `self`（名称是约定，但大家都这么写）
> - 创建实例时 `__init__` 的参数就是你 `类名(...)` 里传的参数
> - 本项目只用了一个类，其他都是函数——说明**函数优先，类按需使用**
